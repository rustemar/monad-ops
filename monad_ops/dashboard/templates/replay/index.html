Stress-event replay archive

Per-event aggregate of retry rate, top retried contracts, base-fee response, validator-timeout %. One link per stress window so anyone reviewing can pull the same numbers without looking up epoch boundaries. Source is a single EU full node — treat it as one viewpoint among many.

Convention

Each Foundation testnet stress event gets a stable URL on this site within ~30 minutes of the window closing:

  • {{ base_url }}/replay/EVENT_ID — HTML landing with summary metrics and a per-batch breakdown.
  • {{ base_url }}/api/window_summary?from_ts_ms=…&to_ts_ms=… — JSON aggregate behind every landing page, also usable standalone.
  • monad-ops replay-export — the CLI that produces a static asset dump of any window. Shipped in Sprint 2; runs against a local state.db so the artefact survives the live server.

Scope

  • Single-viewpoint. All counters are what one full node observed. Other validators may have seen different reorg / retry / timeout signatures during the same window — pair this with a reference RPC for chain-wide truth.
  • Static once written. Replay artefacts don't update after the event closes; if a metric is wrong, the fix is a fresh dump, not in-place patch.
  • Not Foundation guidance. Curated by the operator of this node. Use for retrospective comparison, not as the canonical record.
